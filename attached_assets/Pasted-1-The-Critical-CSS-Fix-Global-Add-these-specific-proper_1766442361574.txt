1. The Critical CSS Fix (Global)

Add these specific properties to your index.css. This tells iOS: "The background is part of the phone, only the content is part of the app."

CSS
/* 1. Kill the bounce on the root */
html, body {
  margin: 0;
  padding: 0;
  width: 100%;
  height: 100%;
  overflow: hidden; /* This is key: prevents the whole page from sliding */
  position: fixed; /* Extreme measure for iOS rubber-banding */
  background-color: #000000;
}

/* 2. Create the animation you requested */
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.animate-fade-in {
  animation: fadeInUp 0.8s ease-out forwards;
}
2. The Component Fix

The reason the previous version might have felt "detached" is that height: 100vh on iOS includes the area under the address bar/home bar. We need to use h-screen or 100% more strategically.

TypeScript
export default function Landing() {
  const [, setLocation] = useLocation();

  return (
    // The "Canvas": Completely fixed and unmovable
    <div style={{ 
      position: 'fixed', 
      inset: 0, 
      overflow: 'hidden', 
      backgroundColor: '#000000' 
    }}>
      
      {/* LAYER 1: The Static Background */}
      <div 
        style={{
          position: 'absolute',
          inset: 0,
          zIndex: 0,
          backgroundImage: `url(${backgroundImage})`,
          backgroundSize: 'cover',
          backgroundPosition: 'center',
          backgroundRepeat: 'no-repeat',
          // This prevents the user from "grabbing" the image and dragging it
          userSelect: 'none',
          pointerEvents: 'none' 
        }}
      />

      {/* LAYER 2: The UI Layer */}
      <div 
        className="animate-fade-in"
        style={{
          position: 'relative',
          zIndex: 1,
          width: '100%',
          height: '100%',
          display: 'grid',
          gridTemplateRows: '1fr auto 1fr auto',
          alignItems: 'center',
          justifyItems: 'center',
          paddingTop: 'env(safe-area-inset-top)',
          paddingBottom: 'env(safe-area-inset-bottom)',
          // touchAction: 'none' stops the "rubber band" drag entirely
          touchAction: 'none' 
        }}
      >
        <div /> {/* Spacer */}

        <img 
          src={logo} 
          alt="BoxStat Logo" 
          style={{ width: '180px', height: 'auto', filter: 'drop-shadow(0 10px 20px rgba(0,0,0,0.5))' }}
        />

        <div /> {/* Spacer */}

        <div style={{ 
          display: 'flex', 
          flexDirection: 'column', 
          alignItems: 'center', 
          gap: '16px', 
          paddingBottom: 'max(60px, env(safe-area-inset-bottom))',
          width: '100%' 
        }}>
          <Button size="lg" onClick={() => setLocation('/registration')} /* styles... */>
            LET'S GO
          </Button>
          <p style={{ color: 'white' }}>...</p>
        </div>
      </div>
    </div>
  );
}
3. Capacitor Specific Fix

In your capacitor.config.ts, you have backgroundColor: '#000000', which is great. However, to truly stop the "white gap" or "gray gap" during a scroll, you should also ensure your web server configuration isn't allowing the webview to bounce.

If you are using the Capacitor Keyboard plugin, adding this to your capacitor.config.ts helps keep the layout from shifting:

TypeScript
Keyboard: {
  resize: 'none', // Prevents the whole screen from "shoving up" when the keyboard appears
  style: 'DARK',
},
