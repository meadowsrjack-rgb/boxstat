Here is the complete solution. I have split this into two parts: the CSS hacks (to fix the browser behavior) and the React Component (to fix the layout and sizing).

Part 1: The CSS (safari-hacks.css)

Add these rules to your global stylesheet (e.g., index.css or globals.css). This is the "Safety Net" that prevents the white gaps even if the content bounces.

CSS
/* 1. Prevent the "White Flash" on rubber-band scroll */
html, body {
  background-color: #000000; /* Must match your landing page background color */
  overscroll-behavior-y: none; /* Disables the vertical bounce/rubber-banding */
  margin: 0;
  padding: 0;
  width: 100%;
  height: 100%;
}

/* 2. Remove the gray highlight box when tapping buttons on iOS */
* {
  -webkit-tap-highlight-color: transparent;
}

/* 3. Ensure the root container handles height correctly */
#root {
  min-height: 100dvh; /* Dynamic Viewport Height */
  display: flex;
  flex-direction: column;
}
Part 2: The Component (Landing.jsx)

I have updated this to use min-h-[100dvh] to fix the address bar resizing issue and cleaned up the structure to ensure the safe areas apply correctly.

JavaScript
import { Button } from "@/components/ui/button";
import { useLocation } from "wouter";
import backgroundImage from "@assets/landing-background.png";
import logo from "@assets/logo2_1762477206651.png";

export default function Landing() {
  const [, setLocation] = useLocation();

  return (
    <>
      {/* BACKGROUND LAYER 
        - Fixed position: It never moves, even if the user manages to scroll.
        - z-0: Sits behind everything.
        - bg-black: Fallback if image loads slow.
      */}
      <div 
        className="fixed inset-0 w-full h-full z-0 pointer-events-none"
        style={{
          backgroundImage: `url(${backgroundImage})`,
          backgroundSize: 'cover',
          backgroundPosition: 'center center',
          backgroundRepeat: 'no-repeat',
          backgroundColor: '#000000',
        }}
      />

      {/* CONTENT LAYER
        - min-h-[100dvh]: Adapts instantly when Safari address bar expands/contracts.
        - overflow-hidden: Prevents accidental horizontal scrolling.
      */}
      <div 
        className="relative z-10 w-full min-h-[100dvh] flex flex-col items-center overflow-hidden"
        style={{
          // These use CSS environment variables to respect the Notch and Home Indicator
          paddingTop: 'calc(2rem + env(safe-area-inset-top, 20px))', 
          paddingBottom: 'calc(1.5rem + env(safe-area-inset-bottom, 20px))',
        }}
      >
        
        {/* Logo Section */}
        <div className="flex-shrink-0 pt-4 animate-in fade-in zoom-in duration-500">
          <img 
            src={logo} 
            alt="BoxStat Logo" 
            className="w-48 h-auto sm:w-64 md:w-80 drop-shadow-2xl"
            data-testid="img-logo"
          />
        </div>

        {/* Spacer - pushes content apart */}
        <div className="flex-1"></div>

        {/* Action Section */}
        <div className="px-4 sm:px-6 lg:px-8 text-center space-y-5 w-full flex-shrink-0 pb-4">
          
          {/* Main Button */}
          <Button 
            size="lg" 
            onClick={() => setLocation('/registration')}
            className="bg-black/20 hover:bg-red-600 text-white font-bold px-12 py-6 rounded-xl shadow-2xl backdrop-blur-sm border border-white/10 transition-all duration-300 transform hover:scale-105 active:scale-95 min-w-[280px] text-[14px] tracking-wider"
            data-testid="button-lets-go"
          >
            LET'S GO
          </Button>

          {/* Sign In Link */}
          <div className="text-white text-sm font-medium tracking-wide">
            <span className="opacity-80">HAVE AN ACCOUNT? </span>
            <button 
              onClick={() => setLocation('/login')}
              className="text-white ml-2 font-bold underline underline-offset-4 hover:text-red-400 transition-colors"
              data-testid="button-sign-in"
            >
              SIGN IN
            </button>
          </div>
        </div>
      </div>
    </>
  );
}
Vital Check: index.html

For any of the env(safe-area-inset-top) padding to work, ensure your index.html head tag has this exact meta tag. If you miss viewport-fit=cover, you will get white bars on the notch.

HTML
<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=0, viewport-fit=cover" />
Summary of Fixes Applied:

Address Bar Bleed: Switched min-h-full to min-h-[100dvh].

White Gaps: Added bg-black fallback to the background div.

Rubber Banding: Added overscroll-behavior-y: none to the CSS.

Touch Feel: Added active:scale-95 to the button for better tactile feedback on mobile.
