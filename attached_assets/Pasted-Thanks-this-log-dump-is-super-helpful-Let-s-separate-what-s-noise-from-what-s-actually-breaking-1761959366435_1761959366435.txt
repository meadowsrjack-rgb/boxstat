## üõ†Ô∏è How to Fix It

### ‚úÖ 1. Mount GooglePlacesAutocomplete *only when the form/modal is visible*

Inside your event form modal (Radix `<Dialog>` or similar):

```tsx
{isDialogOpen && (
  <GooglePlacesAutocomplete
    apiKey={import.meta.env.VITE_GOOGLE_MAPS_API_KEY}
    selectProps={{
      placeholder: "Search location",
      onChange: (place) => {
        // Defensive: handle missing geometry
        const { formatted_address, geometry } = place.value;
        const lat = geometry?.location?.lat();
        const lng = geometry?.location?.lng();
        setFormData((f) => ({
          ...f,
          location: formatted_address || "",
          latitude: lat,
          longitude: lng,
        }));
      },
    }}
  />
)}
```

This guarantees the autocomplete attaches correctly once the modal DOM exists.
