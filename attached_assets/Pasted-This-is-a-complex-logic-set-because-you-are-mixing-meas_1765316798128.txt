To handle this in a Replit app (likely React or similar frontend), you should **not** just list all fields at once. You need a **Dynamic Form** that changes inputs based on the "Trigger Type" selected.

### 2\. The UI Layout (The Window)

You should divide the window into two sections: **Visuals** (Static) and **Logic** (Dynamic).

#### Part A: Visual Identity (Always Visible)

  * **Name Input:** (e.g., "Training Titan")
  * **Tier Selector:** Dropdown [Gold, Purple, Blue, Green, Grey, Special]
  * **Icon Upload:** Image picker.

#### Part B: The Logic Engine (Dynamic)

Create a dropdown labeled **"How is this earned?"** (This is the `Trigger Category`).
Based on what the user selects here, the fields below must change.

**Scenario 1: User selects "Check-in" or "RSVP"**

  * *Used for: Immortal Ironman, Practice Centurion, The Planner.*
  * **Field 1 (Event Filter):** Dropdown [Game, Practice, Skills, FNH, Any].
  * **Field 2 (Mode):** Toggle [Total Cumulative] vs [Consecutive Streak].
  * **Field 3 (Goal):** Number Input (e.g., "50").

**Scenario 2: User selects "System" (Meta-Badges)**

  * *Used for: Superstar Supreme, The Completer, Elite MVP.*
  * *Logic:* This counts how many of *another* badge the user has.
  * **Field 1 (Target Badge):** A search/dropdown of your *existing* badges. (e.g., User selects "MVP").
  * **Field 2 (Goal):** Number Input (e.g., "10").
      * *Result:* "Award this badge when user has 10 of [MVP Badge]."

**Scenario 3: User selects "Time"**

  * *Used for: Dynasty Member, First Year Anniversary.*
  * **Field 1 (Duration):** Number Input (e.g., "5").
  * **Field 2 (Unit):** Dropdown [Years, Months, Days].

**Scenario 4: User selects "Store Purchase"**

  * *Used for: UYP Foundation Program.*
  * **Field 1 (Product SKU):** Input field or dropdown of store items.

**Scenario 5: User selects "Manual"**

  * *Used for: Coachâ€™s Award, MVP (Season Trophy).*
  * *Display:* "No automatic triggers. This badge is assigned by admins/coaches only."
